## Description

Create a reusable Select component in `presentation/components/ui/Select.tsx` for dropdown selections. The component must follow project conventions and include full accessibility support with proper keyboard navigation, ARIA attributes, and screen reader announcements.

## User Story

As a developer, I want a reusable Select component with label and error state so that I can build consistent forms with dropdown selections across the application.

## Acceptance Criteria

1. **Component Structure**
   - [ ] Select component created in `presentation/components/ui/Select.tsx`
   - [ ] Component follows project conventions (arrow function, export default, `type` for props)
   - [ ] Component accepts props: `label`, `name`, `value`, `onChange`, `options`, `placeholder`, `error`, `helperText`, `required`, `disabled`, `ariaLabel`
   - [ ] Component uses SCSS variables from `styles/variables/*` for all styling values
   - [ ] Component SCSS file created in `src/styles/components/ui/Select.module.scss`

2. **Options Support**
   - [ ] Accepts array of options with `value` and `label` properties
   - [ ] Supports option groups (optional, can be added later)
   - [ ] Placeholder option when no value selected
   - [ ] Options are properly rendered in dropdown

3. **Label Support**
   - [ ] Label element properly linked with `htmlFor` and select `id`
   - [ ] Label is visually associated with select field
   - [ ] Required indicator shown when `required` prop is true (e.g., asterisk)
   - [ ] Label text uses i18n translation keys (not hardcoded strings)

4. **Error State**
   - [ ] Error message displayed when `error` prop is provided
   - [ ] Error message uses `role="alert"` for screen readers
   - [ ] Error message is associated with select via `aria-describedby`
   - [ ] Select field has `aria-invalid="true"` when error exists
   - [ ] Error state has distinct visual styling (consistent with Input component)
   - [ ] Error message text uses i18n translation keys

5. **Helper Text**
   - [ ] Helper text displayed when `helperText` prop is provided (no error)
   - [ ] Helper text is associated with select via `aria-describedby`
   - [ ] Helper text uses subtle styling (consistent with Input component)
   - [ ] Helper text text uses i18n translation keys

6. **Accessibility**
   - [ ] Proper `<label>` element linked with `htmlFor` and select `id`
   - [ ] `aria-label` or `aria-labelledby` for all selects
   - [ ] `aria-required="true"` for required fields
   - [ ] `aria-disabled="true"` for disabled selects
   - [ ] `aria-invalid="true"` and `aria-describedby` for error states
   - [ ] Keyboard navigation: Arrow keys navigate options, Enter/Space selects, Escape closes
   - [ ] Focus indicators: visible focus outline
   - [ ] Proper `role="combobox"` or native select with proper ARIA (depending on implementation)
   - [ ] Accessibility IDs use centralized utilities from `shared/a11y/`

7. **States**
   - [ ] Default state: normal select appearance
   - [ ] Focus state: visual feedback when focused
   - [ ] Error state: distinct styling when error exists
   - [ ] Disabled state: visually distinct, non-interactive
   - [ ] Open state (if custom dropdown): visual indication when dropdown is open

8. **Styling**
   - [ ] All colors use SCSS variables (no hardcoded values)
   - [ ] All spacing uses SCSS variables
   - [ ] All font sizes use SCSS variables
   - [ ] Uses BEM methodology for class names (kebab-case, `__` for elements, `--` for modifiers)
   - [ ] No inline CSS
   - [ ] No `!important`
   - [ ] Consistent styling with Input component
   - [ ] Dropdown menu styling (if custom dropdown implementation)

9. **TypeScript**
   - [ ] Props typed with `type` (not `interface`)
   - [ ] Explicit return type for component
   - [ ] Proper TypeScript types for all props
   - [ ] No `any` types
   - [ ] Type for options: `{ value: string; label: string }[]`
   - [ ] Proper event types (`React.ChangeEvent<HTMLSelectElement>`)

10. **i18n Integration**
    - [ ] All user-facing strings use `useTranslation()` hook
    - [ ] Translation keys added to `src/shared/i18n/messages/fr.json`
    - [ ] Translation keys follow naming conventions
