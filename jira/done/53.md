## Description

Create a reusable Button component in `presentation/components/ui/Button.tsx` with primary, secondary, and danger variants. The component must follow project conventions (arrow function, export default, SCSS variables) and include full accessibility support with proper ARIA attributes and keyboard navigation.

## User Story

As a developer, I want a reusable Button component with variants so that I can consistently style interactive buttons across the application without duplicating code.

## Acceptance Criteria

1. **Component Structure**
   - [ ] Button component created in `presentation/components/ui/Button.tsx`
   - [ ] Component follows project conventions (arrow function, export default, `type` for props)
   - [ ] Component accepts props: `label`, `onClick`, `variant`, `disabled`, `type`, `ariaLabel`
   - [ ] Component uses SCSS variables from `styles/variables/*` for all styling values
   - [ ] Component SCSS file created in `src/styles/components/ui/Button.module.scss`

2. **Variants**
   - [ ] Primary variant (default): emphasizes primary actions
   - [ ] Secondary variant: for secondary actions
   - [ ] Danger variant: for destructive actions
   - [ ] Variants use distinct colors from SCSS variables
   - [ ] Variants have consistent styling and spacing

3. **States**
   - [ ] Default state: normal button appearance
   - [ ] Hover state: visual feedback on hover
   - [ ] Active state: visual feedback when clicked
   - [ ] Disabled state: visually distinct, non-interactive
   - [ ] Loading state: shows spinner when loading (optional for future enhancement)

4. **Accessibility**
   - [ ] Proper `role="button"` attribute
   - [ ] `aria-label` or `aria-labelledby` for all buttons
   - [ ] Keyboard navigation: Enter and Space keys trigger onClick
   - [ ] Focus indicators: visible focus outline
   - [ ] Disabled state: `aria-disabled="true"` and proper keyboard handling
   - [ ] Accessibility IDs use centralized utilities from `shared/a11y/`

5. **Styling**
   - [ ] All colors use SCSS variables (no hardcoded values)
   - [ ] All spacing uses SCSS variables
   - [ ] All font sizes use SCSS variables
   - [ ] Uses BEM methodology for class names (kebab-case, `__` for elements, `--` for modifiers)
   - [ ] No inline CSS
   - [ ] No `!important`

6. **TypeScript**
   - [ ] Props typed with `type` (not `interface`)
   - [ ] Explicit return type for component
   - [ ] Proper TypeScript types for all props
   - [ ] No `any` types

7. **Component Export**
   - [ ] Component exported as default
   - [ ] Component can be imported from `@/presentation/components/ui/Button`

## Technical Considerations
