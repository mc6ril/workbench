## Description

Create a reusable Checkbox component in `presentation/components/ui/Checkbox.tsx` with label support. The component must follow project conventions and include full accessibility support with proper form labeling and ARIA attributes.

## User Story

As a developer, I want a reusable Checkbox component with label so that I can build consistent forms with checkbox inputs across the application.

## Acceptance Criteria

1. **Component Structure**
   - [ ] Checkbox component created in `presentation/components/ui/Checkbox.tsx`
   - [ ] Component follows project conventions (arrow function, export default, `type` for props)
   - [ ] Component accepts props: `label`, `name`, `checked`, `onChange`, `required`, `disabled`, `indeterminate` (optional), `ariaLabel`
   - [ ] Component uses SCSS variables from `styles/variables/*` for all styling values
   - [ ] Component SCSS file created in `src/styles/components/ui/Checkbox.module.scss`

2. **Label Support**
   - [ ] Label element properly linked with `htmlFor` and checkbox `id`
   - [ ] Label is visually associated with checkbox
   - [ ] Label appears after checkbox (standard pattern)
   - [ ] Clicking label toggles checkbox
   - [ ] Label text uses i18n translation keys (not hardcoded strings)

3. **Checkbox States**
   - [ ] Unchecked state: empty checkbox
   - [ ] Checked state: checkbox with checkmark
   - [ ] Indeterminate state (optional): partial checkmark for "some selected" scenarios
   - [ ] Disabled state: visually distinct, non-interactive
   - [ ] Focus state: visible focus outline

4. **Accessibility**
   - [ ] Proper `<label>` element linked with `htmlFor` and checkbox `id`
   - [ ] `aria-label` or `aria-labelledby` for all checkboxes (when label is not visible)
   - [ ] `aria-required="true"` for required checkboxes
   - [ ] `aria-disabled="true"` for disabled checkboxes
   - [ ] `aria-checked="true"`, `aria-checked="false"`, or `aria-checked="mixed"` (for indeterminate)
   - [ ] Keyboard navigation: Space key toggles checkbox, Tab key moves focus
   - [ ] Focus indicators: visible focus outline
   - [ ] Accessibility IDs use centralized utilities from `shared/a11y/`

5. **Styling**
   - [ ] All colors use SCSS variables (no hardcoded values)
   - [ ] All spacing uses SCSS variables
   - [ ] All font sizes uses SCSS variables
   - [ ] Custom checkbox styling (replaces browser default)
   - [ ] Checkmark icon or visual indicator
   - [ ] Uses BEM methodology for class names (kebab-case, `__` for elements, `--` for modifiers)
   - [ ] No inline CSS
   - [ ] No `!important`
   - [ ] Consistent styling with other form components

6. **TypeScript**
   - [ ] Props typed with `type` (not `interface`)
   - [ ] Explicit return type for component
   - [ ] Proper TypeScript types for all props
   - [ ] No `any` types
   - [ ] Proper event types (`React.ChangeEvent<HTMLInputElement>`)

7. **i18n Integration**
   - [ ] Label text uses `useTranslation()` hook
   - [ ] Translation keys added to `src/shared/i18n/messages/fr.json`
   - [ ] Translation keys follow naming conventions

8. **Component Export**
   - [ ] Component exported as default
   - [ ] Component can be imported from `@/presentation/components/ui/Checkbox`
